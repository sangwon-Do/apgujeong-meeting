압구정 B2B 운영조건 점검 브리프

현대건설 차주 미팅용 외부 공유 초안입니다. 공개 GitHub 기준으로 확인 가능한 운영 근거와 현장/정책 협의가 필요한 공백을 분리해, 이번 미팅에서 반드시 확정해야 할 의사결정 항목만 빠르게 정리했습니다.

이번 문서의 전제
  • 검토 기준은 공개 저장소 `hkmc-airlab/shucle-product`의 현재 README, 2020 README, 최근 요구사항 템플릿입니다.
  • 공개 텍스트 자산과 GitHub 이슈 검색 기준으로 `압구정`, `현대건설` 직접 표기 초안은 확인되지 않았습니다.
  • 따라서 본 문서는 공개 B2B 운영기획 축을 근거로 삼고, 압구정 미팅에서 확정해야 할 실제 운영안 항목을 정리합니다.

한눈에 보는 판단 기준

이번 미팅 자료는 "공개 제품기획 문서로 확인 가능한 운영 축"과 "현장/정책 협의 없이는 확정할 수 없는 항목" 을 분리해 보는 것이 핵심입니다.

공개 리포지토리에서 확인된 운영 근거

`shucle-product`의 히스토리에는 B2B 플랫폼 정책, 차량 계획, 정산, 플릿관리, 운영 매뉴얼, 서비스지역과 픽업/드롭오프 포인트 관리가 명시돼 있습니다. 최근에는 운영사·운수사 요구사항을 이슈 템플릿으로 구조화해 지역별 운영요건을 기록하도록 정비한 흔적도 확인됩니다.

B2B 운영정책과 플릿관리 축은 제품 기획에 명시됨 확인 가능
운영사 요구사항을 지역/프로젝트 단위로 적재하는 구조 존재 확인 가능
서비스지역·픽업/드롭오프 포인트 관리 축이 명시됨 확인 가능

공개 리포지토리에서 바로 확인되지 않는 항목

압구정 전용 운영시간표, 현대건설 협의 메모, 휴차지 후보지, 가상정류장 좌표안은 공개 텍스트 자산에 없습니다. 이 항목은 이번 미팅에서 사업지 운영표준으로 직접 확정해야 합니다.

안건별 점검 포인트

각 안건마다 "공개 자료 근거", "이번 미팅에서 물어야 할 질문", "결정 산출물"을 붙였습니다.

1. 서비스 운영시간 확정

공개 기획 문서에는 운영사 요구사항을 `정책 변경`, `운영 효율화`, `서비스지역·노선` 카테고리로 기록하는 구조가 있습니다. 즉, 운영시간은 제품 기능보다 운영정책 요건으로 정리해 확정하는 방식이 맞습니다.

공개 자료 근거

`운영사·운수사 요구사항 등록`, `정책 변경`, `정기미팅`, `프로젝트`, `지역`

  • 주중/주말 동일 운영인지, 시간대 분리 운영인지
  • 첫차/막차 기준이 서비스 노출 시간인지 실제 운행 가능 시간인지
  • 야간 제한 시 호출 차단 시점과 탑승 종료 허용 시점을 어떻게 나눌지

2. 패널티 정책

과거 기획 문서에는 `cost settlement`, `fleet management`, `Operation manual`이 명시됩니다. 즉 패널티는 단순 기능 항목보다 운영정산 기준과 운영 매뉴얼 안에서 확정해야 하는 성격이 강합니다.

공개 자료 근거

`B2B : vehicle plan, cost settlement, fleet management`, `CS manual`, `Operation manual`

  • 운영시간 외 반납, 지정 외 구역 반납, 휴차지 외 적치에 대해 각각 별도 패널티를 둘지
  • 현장 확인 후 부과인지 시스템 자동 부과인지
  • 패널티 면책 기준과 사진 증빙 책임 주체가 누구인지

3. 휴차지 장소 및 스케줄

공개 자료에는 휴차지 리스트 자체는 없지만, 운영사 요건을 프로젝트/지역 단위로 적재하는 구조가 이미 있습니다. 따라서 휴차지는 시스템 화면보다 운영 매뉴얼과 현장 운영표로 확정해야 하는 영역입니다.

공개 자료 근거

`요구사항 등록`, `프로젝트`, `지역`, `운영 효율화`, `운영사 또는 운수사로부터 인입된 요건`

  • 휴차지 후보지의 정확한 주소, 수용 대수, 접근 가능 시간
  • 회수/재배치 요일 및 시간, 비상 상황 시 대체 장소
  • 현대건설 측 승인 프로세스와 현장 담당자 연락 체계

4. 가상정류장 현장 위치 확인

2020 기획 문서에는 `service zone`, `pickup/dropoff points`, `walk network`, `POI`가 명시돼 있습니다. 즉 가상정류장은 제품 구조상 새로운 개념이 아니라, 지역별 포인트 설계와 운영정책의 조합으로 보는 것이 맞습니다.

공개 자료 근거

`service zone`, `pickup/dropoff points`, `walk network`, `POI`

  • 후보 위치별 보행 동선, 차량 동선, 안전 이격거리 확인
  • 승차 가능 지점과 하차 가능 지점을 동일 좌표로 둘지 분리할지
  • 실제 운영구역 반영 시 예외 구역이나 속도 제한을 둘 필요가 있는지

리포지토리 히스토리 요약

압구정 직접 초안은 보이지 않았고, 대신 B2B 운영기획 구조가 어떻게 진화했는지를 추렸습니다.

2020

B2B 기획 축 등장

README_2020에서 B2B 플랫폼 정책을 `vehicle plan`, `cost settlement`, `fleet management`로 정의했습니다.

2020

운영 매뉴얼 축 확인

`Service platform CRM` 항목에 B2B 대상 `CS manual`, `Operation manual`이 잡혀 있어 운영표준 문서화 방향이 보입니다.

2020

지역 포인트 관리 축 확인

`service zone`, `pickup/dropoff points`, `walk network`, `POI`가 함께 적혀 있어 가상정류장 검토의 상위 프레임을 제공합니다.

2025-2026

요구사항 관리 체계 정비

최근에는 운영사·운수사 요구사항 등록 템플릿과 프로젝트 필드 동기화 워크플로가 추가돼, 사업지별 운영요건을 이슈 구조로 다루는 체계가 강화됐습니다.

현재까지 확인한 공개 근거

외부 미팅에서 말해도 되는 수준으로, 공개 저장소 기준으로 확인 가능한 문구만 정리했습니다.

출처 확인 문구 미팅 자료에서의 해석

`README_2020` `CS manual`, `Operation manual` 휴차지와 현장 운영 프로세스는 매뉴얼 관점으로 정리하는 편이 자연스러움

회의 직후 남겨야 할 결정 사항

의견 교환으로 끝나지 않도록, 회의 직후 문서화할 항목을 미리 비워 두었습니다.

Decision 01

운영시간 확정안

예: 평일 06:30-22:30 / 주말 08:00-22:00, 마지막 호출 마감 21:45

Decision 02

패널티 적용표

예: 지정 외 반납 / 운영시간 외 반납 / 휴차지 미이동 각각의 금액, 증빙, 면책 기준

Decision 03

휴차지 운영표

예: 위치, 수용 대수, 회수 시간, 현장 담당자, 비상시 대체 장소

Decision 04

가상정류장 좌표 시트

예: 후보 위치별 좌표, 사진, 안전 이슈, 최종 승인 여부
